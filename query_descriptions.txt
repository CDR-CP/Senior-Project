 - Energy consumption per run (battery health analysis): 
Compute instantaneous power (voltage × current) and integrate over time to estimate total energy used per run in watt-hours. 
This gives a concrete performance metric and lets us compare efficiency across runs using run_id.
     - This is done by computing instantaneous power for every row, 
       computing the change in seconds between that row and the previous,
       then getting instantaneous power per change in seconds, and converting to watt-hours

 - Dive profile and phase segmentation: 
Use depth_m over time to identify descent, steady-state depth, and ascent phases. 
From this, compute maximum depth, average depth, descent/ascent rates, and time spent at depth.
     - This is done by calulating change in vertical meters per second between subsequent rows, and 
       the computing per run stats regarding max and avg depth, change in depth, time spent near max depth.

 - Voltage sag and high-load event detection: 
Detect moments where battery_voltage_v drops sharply while battery_current_a spikes. 
This reveals stress events on the power system and ties electrical behavior to operational activity.
     - This is done by computing change in voltage and change in current between rows, 
       then filtering for rows where change in voltage drops while change in current rises.

 - Motion intensity and anomaly detection from IMU: 
Compute total acceleration magnitude √(ax² + ay² + az²), apply rolling statistics, and flag spikes or abnormal variance. 
This helps identify aggressive maneuvers, instability, or impacts.
     - This is done by calculating acceleration magnitude per row, 
       taking rolling mean and rolling std deviation, using those to compute z-score for every row, 
       then filtering for rows with a z-score >= 3 to highlight rows where 
       the acceleration magnitude suddenly spikes or drops outside of whats normal for recent rows.
